## Java Fundamentals (50 Questions)

### Q1: What is JVM?
**A:** Java Virtual Machine - executes Java bytecode, provides platform independence, manages memory automatically.

### Q2: Difference between JDK, JRE, and JVM?
**A:** 
- JDK = Development Kit (compiler + tools + JRE)
- JRE = Runtime Environment (JVM + libraries)
- JVM = Virtual Machine (executes bytecode)

### Q3: What is bytecode?
**A:** Intermediate code generated by javac compiler (.class files), executed by JVM, platform-independent.

### Q4: Explain public static void main(String[] args)
**A:**
- `public`: accessible from anywhere
- `static`: can call without creating object
- `void`: returns nothing
- `main`: entry point
- `String[] args`: command-line arguments

### Q5: What are the 8 primitive types?
**A:** byte, short, int, long, float, double, char, boolean

### Q6: Difference between == and equals()?
**A:**
- `==`: compares references (memory address)
- `equals()`: compares content/values

### Q7: What is autoboxing/unboxing?
**A:**
- Autoboxing: automatic conversion primitive â†’ wrapper (int â†’ Integer)
- Unboxing: automatic conversion wrapper â†’ primitive (Integer â†’ int)

### Q8: Explain String immutability
**A:** String objects cannot be modified after creation. Any modification creates new String object. Benefits: thread-safe, string pool, security.

### Q9: String vs StringBuilder vs StringBuffer?
**A:**
- String: immutable, thread-safe
- StringBuilder: mutable, not thread-safe, faster
- StringBuffer: mutable, thread-safe, slower

### Q10: What is String pool?
**A:** Special memory region in heap where String literals are stored. Same literals share same object, saves memory.

### Q11-20: OOP Questions

### Q11: What are the 4 OOP principles?
**A:**
1. Encapsulation (data hiding)
2. Inheritance (code reuse)
3. Polymorphism (many forms)
4. Abstraction (hiding complexity)

### Q12: Difference between abstract class and interface?
**A:**
- Abstract class: can have concrete methods, single inheritance, can have state
- Interface: all methods abstract (before Java 8), multiple inheritance, no state

### Q13: Can interface have concrete methods?
**A:** Yes (Java 8+): default methods and static methods

### Q14: What is method overloading?
**A:** Same method name, different parameters (number/type/order). Compile-time polymorphism.

### Q15: What is method overriding?
**A:** Subclass provides specific implementation of superclass method. Same signature. Runtime polymorphism.

### Q16: Difference between overloading and overriding?
**A:**
- Overloading: same class, different signature, compile-time
- Overriding: inheritance, same signature, runtime

### Q17: What is super keyword?
**A:** Refers to immediate parent class. Used to: call parent constructor, access parent methods/fields.

### Q18: What is this keyword?
**A:** Refers to current object. Used to: differentiate instance variable from parameter, call another constructor, pass current object.

### Q19: Can we override static methods?
**A:** No. Static methods belong to class, not object. It's method hiding, not overriding.

### Q20: Can we override private methods?
**A:** No. Private methods are not visible to subclasses.

### Q21-30: Exception Handling

### Q21: Checked vs Unchecked exceptions?
**A:**
- Checked: must handle (IOException, SQLException), compile-time
- Unchecked: optional handling (NullPointerException, ArrayIndexOutOfBoundsException), runtime

### Q22: Exception hierarchy?
**A:** Throwable â†’ Error / Exception â†’ RuntimeException / Checked Exceptions

### Q23: try-catch-finally execution order?
**A:** try â†’ catch (if exception) â†’ finally (always executes)

### Q24: Can finally block be skipped?
**A:** Only if: System.exit(), JVM crash, or infinite loop in try/catch

### Q25: try-with-resources?
**A:** Automatic resource management. Closes resources automatically. Resource must implement AutoCloseable.

### Q26: throw vs throws?
**A:**
- throw: throws exception explicitly (in method body)
- throws: declares exception (in method signature)

### Q27: Can we have try without catch?
**A:** Yes, if finally block or try-with-resources present.

### Q28: Custom exception creation?
**A:** Extend Exception (checked) or RuntimeException (unchecked)

### Q29: Multi-catch block?
**A:** Java 7+: `catch (IOException | SQLException e)`

### Q30: Exception chaining?
**A:** Attach one exception to another using constructors or initCause()

### Q31-40: Collections

### Q31: Collection vs Collections?
**A:**
- Collection: interface (root of collection hierarchy)
- Collections: utility class (sort, shuffle, etc.)

### Q32: List vs Set vs Map?
**A:**
- List: ordered, duplicates allowed
- Set: unordered, no duplicates
- Map: key-value pairs, no duplicate keys

### Q33: ArrayList vs LinkedList?
**A:**
- ArrayList: dynamic array, fast random access O(1), slow insertion/deletion O(n)
- LinkedList: doubly linked list, slow access O(n), fast insertion/deletion O(1)

### Q34: HashMap vs TreeMap vs LinkedHashMap?
**A:**
- HashMap: unordered, O(1), null key allowed
- TreeMap: sorted by keys, O(log n), no null keys
- LinkedHashMap: insertion order, O(1), null key allowed

### Q35: HashSet implementation?
**A:** Backed by HashMap. Elements stored as keys with dummy value.

### Q36: How HashMap works?
**A:** Uses hashing. Stores in buckets. hashCode() determines bucket, equals() handles collisions.

### Q37: HashMap collision handling?
**A:** Chaining (linked list). Java 8+: tree if chain >8 elements.

### Q38: Comparable vs Comparator?
**A:**
- Comparable: natural ordering, implements compareTo(), modifies class
- Comparator: custom ordering, implements compare(), separate class

### Q39: fail-fast vs fail-safe?
**A:**
- fail-fast: throws ConcurrentModificationException if modified during iteration
- fail-safe: works on copy, no exception

### Q40: Iterator vs ListIterator?
**A:**
- Iterator: forward only, for all collections
- ListIterator: bidirectional, only for List

### Q41-50: Generics & Misc

### Q41: What are Generics?
**A:** Type safety at compile-time. Eliminates casting. Example: `List<String>`

### Q42: Type erasure?
**A:** Generic type information removed at runtime. Ensures backward compatibility.

### Q43: Bounded types?
**A:** `<T extends Number>` (upper bound), `<T super Integer>` (lower bound)

### Q44: Wildcards in Generics?
**A:**
- `<?>`: unbounded
- `<? extends T>`: upper bounded (read-only)
- `<? super T>`: lower bounded (write-only)

### Q45: Serialization?
**A:** Converting object to byte stream. Implement Serializable interface. Transient fields excluded.

### Q46: serialVersionUID?
**A:** Version control for serialization. Ensures compatibility between versions.

### Q47: Cloning?
**A:** Creating object copy. Implement Cloneable, override clone(). Shallow vs deep copy.

### Q48: static keyword?
**A:** Belongs to class, not object. Shared across all instances. Can access without object.

### Q49: final keyword?
**A:**
- final variable: constant
- final method: cannot override
- final class: cannot extend

### Q50: finalize() method?
**A:** Called by GC before object destruction. Deprecated in Java 9.

## Modern Java (30 Questions)

### Q51: Lambda expression syntax?
**A:** `(parameters) -> expression` or `(parameters) -> { statements; }`

### Q52: Functional interface?
**A:** Interface with single abstract method. Can use lambda. @FunctionalInterface annotation.

### Q53: Method reference types?
**A:**
1. Static: `ClassName::methodName`
2. Instance: `object::methodName`
3. Constructor: `ClassName::new`

### Q54: Stream API operations?
**A:**
- Intermediate: filter, map, sorted (lazy, return Stream)
- Terminal: collect, forEach, count (eager, trigger execution)

### Q55: map() vs flatMap()?
**A:**
- map(): one-to-one (Stream<R>)
- flatMap(): one-to-many (Stream<Stream<R>> â†’ Stream<R>)

### Q56: Optional benefits?
**A:** Avoids null checks, prevents NullPointerException, makes null explicit.

### Q57: Optional methods?
**A:** isPresent(), ifPresent(), orElse(), orElseThrow(), map(), flatMap()

### Q58: Default method in interface?
**A:** Java 8+ feature. Provides default implementation. Enables backward compatibility.

### Q59: Static method in interface?
**A:** Java 8+. Utility methods. Called via interface name.

### Q60: forEach() vs for loop?
**A:**
- forEach(): functional, cannot break, lambda-based
- for loop: imperative, can break, traditional

### Q61-70: Stream & Lambda Deep Dive

### Q61: Collectors.toList() vs collect()?
**A:** toList() is convenience method. collect() is generic, flexible terminal operation.

### Q62: groupingBy() example?
**A:** `stream.collect(Collectors.groupingBy(Employee::getDepartment))`

### Q63: Parallel streams?
**A:** `.parallelStream()` - uses ForkJoinPool, processes elements in parallel.

### Q64: When to use parallel streams?
**A:** Large datasets (>10K elements), CPU-intensive operations, stateless operations.

### Q65: CompletableFuture?
**A:** Async programming. Chain operations. Non-blocking. Better than Future.

### Q66: thenApply() vs thenCompose()?
**A:**
- thenApply(): transform result
- thenCompose(): chain another CompletableFuture

### Q67: Virtual Threads (Java 21)?
**A:** Lightweight threads. JVM-managed. Enables millions of concurrent threads.

### Q68: Records (Java 14+)?
**A:** Immutable data carriers. Auto-generates constructor, getters, equals, hashCode, toString.

### Q69: Sealed classes (Java 17)?
**A:** Restricts which classes can extend/implement. `sealed class permits`

### Q70: Pattern matching for instanceof?
**A:** `if (obj instanceof String s) { use s directly }`

### Q71-80: Date/Time, Misc

### Q71: LocalDate vs LocalDateTime vs ZonedDateTime?
**A:**
- LocalDate: date only
- LocalDateTime: date + time
- ZonedDateTime: date + time + timezone

### Q72: Duration vs Period?
**A:**
- Duration: time-based (hours, minutes, seconds)
- Period: date-based (years, months, days)

### Q73: var keyword?
**A:** Local variable type inference (Java 10+). Compiler infers type.

### Q74: Switch expressions (Java 14+)?
**A:** `int result = switch(value) { case 1 -> 10; case 2 -> 20; default -> 0; };`

### Q75: Text blocks (Java 15+)?
**A:** Multi-line strings: ```String json = """ { "name": "John" } """;```

## JVM & Performance (20 Questions)

### Q81: JVM memory areas?
**A:** Heap, Stack, Metaspace, Code Cache, PC Register

### Q82: Heap vs Stack?
**A:**
- Heap: objects, GC managed, shared, slower
- Stack: method calls, local variables, thread-specific, faster

### Q83: Garbage Collection?
**A:** Automatic memory management. Removes unreferenced objects. Frees memory.

### Q84: Types of GC?
**A:** Serial, Parallel, CMS, G1GC (default), ZGC, Shenandoah

### Q85: When does GC run?
**A:** When heap is full, System.gc() called (suggestion), JVM decides

### Q86: Memory leak in Java?
**A:** Yes! Unreachable objects still referenced. Common: static collections, listeners, ThreadLocal.

### Q87: -Xms vs -Xmx?
**A:**
- -Xms: initial heap size
- -Xmx: maximum heap size

### Q88: OutOfMemoryError causes?
**A:** Heap full, Metaspace full, unable to create native thread, GC overhead limit exceeded

### Q89: How to analyze memory?
**A:** jmap (heap dump), jconsole, VisualVM, JProfiler, heap dump analysis (MAT)

### Q90: Class loading phases?
**A:** Loading â†’ Linking (Verify â†’ Prepare â†’ Resolve) â†’ Initialization

## Spring Framework (40 Questions)

### Q101: What is Spring?
**A:** Java framework for enterprise applications. Provides DI, AOP, transaction management.

### Q102: Spring vs Spring Boot?
**A:**
- Spring: framework, manual configuration
- Spring Boot: opinionated, auto-configuration, embedded server, starters

### Q103: Dependency Injection?
**A:** IoC design pattern. Spring creates and injects dependencies. Loose coupling.

### Q104: Types of DI?
**A:**
1. Constructor injection (recommended)
2. Setter injection
3. Field injection

### Q105: @Autowired vs @Inject vs @Resource?
**A:**
- @Autowired: Spring-specific, by type
- @Inject: JSR-330, by type
- @Resource: JSR-250, by name

### Q106: Bean scopes?
**A:** singleton (default), prototype, request, session, application

### Q107: @Component vs @Service vs @Repository?
**A:** All create beans. @Service (business logic), @Repository (DAO, exception translation), @Component (generic)

### Q108: @Configuration vs @Component?
**A:** @Configuration: contains @Bean methods. @Component: single bean.

### Q109: ApplicationContext vs BeanFactory?
**A:** ApplicationContext: superset, eager loading, i18n. BeanFactory: basic, lazy loading.

### Q110: Bean lifecycle?
**A:** Instantiate â†’ Populate Properties â†’ setBeanName â†’ setBeanFactory â†’ postProcessBeforeInit â†’ afterPropertiesSet â†’ init-method â†’ postProcessAfterInit â†’ Use â†’ destroy

### Q111-120: Spring MVC & REST

### Q111: @Controller vs @RestController?
**A:**
- @Controller: returns view
- @RestController: @Controller + @ResponseBody, returns data

### Q112: @RequestMapping vs @GetMapping?
**A:** @GetMapping is shortcut for @RequestMapping(method=GET)

### Q113: @RequestBody vs @ResponseBody?
**A:**
- @RequestBody: HTTP request â†’ Java object
- @ResponseBody: Java object â†’ HTTP response

### Q114: @PathVariable vs @RequestParam?
**A:**
- @PathVariable: /users/{id}
- @RequestParam: /users?id=1

### Q115: Exception handling in Spring?
**A:** @ExceptionHandler, @ControllerAdvice, ResponseEntityExceptionHandler

### Q116: How to validate input?
**A:** @Valid, @Validated, @NotNull, @NotBlank, @Size, @Email

### Q117: ResponseEntity?
**A:** Represents HTTP response: status, headers, body. Full control over response.

### Q118: @Transactional?
**A:** Declarative transaction management. Auto commit/rollback. Can be on class or method.

### Q119: Transaction propagation?
**A:** REQUIRED (default), REQUIRES_NEW, MANDATORY, SUPPORTS, NOT_SUPPORTED, NEVER, NESTED

### Q120: AOP in Spring?
**A:** Aspect-Oriented Programming. Cross-cutting concerns. @Aspect, @Before, @After, @Around

### Q121-130: Spring Data & Security

### Q121: JPA vs Hibernate?
**A:** JPA: specification. Hibernate: implementation (ORM framework).

### Q122: @Entity vs @Table?
**A:**
- @Entity: marks JPA entity
- @Table: specifies table name

### Q123: Fetch types?
**A:** EAGER (load immediately), LAZY (load when accessed)

### Q124: @OneToMany vs @ManyToOne?
**A:** Define relationship direction. @OneToMany on parent, @ManyToOne on child.

### Q125: N+1 problem?
**A:** 1 query for list + N queries for each item's associations. Fix: JOIN FETCH, EntityGraph

### Q126: Spring Security filters?
**A:** Chain of filters. UsernamePasswordAuthenticationFilter, BasicAuthenticationFilter, etc.

### Q127: Authentication vs Authorization?
**A:**
- Authentication: verify identity (who are you?)
- Authorization: verify permissions (what can you do?)

### Q128: JWT structure?
**A:** Header.Payload.Signature (Base64 encoded)

### Q129: @PreAuthorize vs @Secured?
**A:**
- @PreAuthorize: SpEL expressions, more flexible
- @Secured: simple roles, less flexible

### Q130: CSRF protection?
**A:** Cross-Site Request Forgery. Spring adds CSRF token. Validate on state-changing requests.

## Microservices (20 Questions)

### Q141: Monolith vs Microservices?
**A:**
- Monolith: single deployable, tight coupling, simple
- Microservices: multiple services, loose coupling, complex

### Q142: Service discovery?

### Q143: API Gateway?

### Q148: Event-driven architecture?

### Q149: CAP theorem?

### Q150: Service mesh?
**A:** Infrastructure layer for service-to-service communication. Istio, Linkerd.

## Databases (20 Questions)

### Q161: SQL vs NoSQL?
**A:**
- SQL: structured, ACID, schema, relational
- NoSQL: flexible schema, BASE, scalable, document/key-value/graph

### Q162: ACID properties?
**A:**
- Atomicity: all or nothing
- Consistency: valid state
- Isolation: concurrent transactions
- Durability: persist after commit

### Q163: Transaction isolation levels?
**A:** READ_UNCOMMITTED, READ_COMMITTED, REPEATABLE_READ, SERIALIZABLE

### Q164: Index types?
**A:** B-Tree, Hash, Full-text, Spatial, Covering

### Q165: When to use index?
**A:** Columns in WHERE, JOIN, ORDER BY. High cardinality. Read-heavy.

### Q166: MongoDB vs PostgreSQL?
**A:**
- MongoDB: document, flexible schema, horizontal scaling
- PostgreSQL: relational, ACID, complex queries, vertical scaling

### Q167: Database sharding?
**A:** Horizontal partitioning. Split data across multiple databases. Scalability.

### Q168: Connection pooling?
**A:** Reuse database connections. HikariCP, C3P0. Improves performance.

### Q169: What is ORM?
**A:** Object-Relational Mapping. Maps objects to database tables. Hibernate, JPA.

### Q170: Lazy loading?
**A:** Load data on demand, not upfront. Prevents N+1 problem if used correctly.

## Docker & Kubernetes (20 Questions)

### Q181: What is Docker?
**A:** Containerization platform. Packages app + dependencies. Isolated, portable.

### Q182: Container vs VM?
**A:**
- Container: shares OS kernel, lightweight, fast
- VM: full OS, heavy, slower

### Q183: Dockerfile commands?
**A:** FROM, RUN, COPY, ADD, CMD, ENTRYPOINT, EXPOSE, ENV, WORKDIR

### Q184: CMD vs ENTRYPOINT?
**A:**
- CMD: default command, can override
- ENTRYPOINT: main command, append args

### Q185: Docker volumes?
**A:** Persist data outside container. Share data between containers.

### Q186: Multi-stage builds?
**A:** Multiple FROM statements. Optimize image size. Build stage + runtime stage.

### Q187: What is Kubernetes?
**A:** Container orchestration. Automates deployment, scaling, management.

### Q188: K8s components?
**A:** Control Plane (API Server, Scheduler, Controller, etcd), Nodes (Pods, Kubelet, Kube-proxy)

### Q189: Pod vs Container?
**A:** Pod: smallest unit, can have multiple containers. Container: single app instance.

### Q190: Deployment vs StatefulSet?
**A:**
- Deployment: stateless, random pod names
- StatefulSet: stateful, stable pod names, ordered
